# what is docker, dockerfile, docker image, docker container

## Docker 

Docker is a containerization tool.

It is used to package an application with its environment.

This helps avoid the problem: “It works on my machine, but not on another machine.”



In simple words, Docker makes applications easier to package, deploy, and run consistently in different environments.

Docker is lighter than a virtual machine because containers share the host machine’s operating system kernel.



Docker 是一种容器化技术，用来快速打包、分发和运行应用。

它可以把应用程序、依赖、配置和运行环境一起封装起来。



## Dockerfile 

A Dockerfile is a text file that contains instructions for building a Docker image. It defines things like the base image, dependencies, files, commands, and application startup process.



Dockerfile 是一个文本文件，里面写着如何构建 Docker 镜像的步骤。

Dockerfile 常见指令包括 FROM、RUN、COPY、WORKDIR、EXPOSE 和 CMD。

执行 `docker build` 根据 Dockerfile 构建出 Docker Image；再执行 `docker run` 基于该 Image 创建并启动一个 Docker Container 实例。





## docker image

A Docker image is a read-only template that contains what the application needs to run.



## docker container

A Docker container is a running instance of an image.

Docker Container（容器）是镜像运行后产生的实例。





# docker vs VM

Docker and virtual machines (VMs) are both technologies used to provide isolated environments for deploying and running applications.

In terms of runtime, Docker runs applications in containers, while a VM runs an operating system and applications inside a virtual machine.

In terms of implementation, Docker shares the host operating system’s kernel, while a VM uses virtualization technology to simulate hardware.

In terms of resource usage, Docker is more lightweight, while VMs require more memory and disk space.

In terms of startup speed, Docker starts faster, while a VM is slower because it needs to boot a complete operating system.

In terms of isolation, VMs generally provide stronger isolation, while Docker containers provide relatively weaker isolation.

In terms of use cases, Docker is better suited for microservices and rapid scaling, while VMs are more suitable for scenarios that require a complete operating system or stronger isolation.



Docker 和 VM 都是用来提供隔离运行环境、部署和运行应用的技术。

在运行形态方面，Docker 以容器运行应用，VM 以虚拟计算机运行操作系统和应用。

在实现方式方面，Docker 共享宿主机内核，VM 通过虚拟化技术模拟硬件。

在资源占用方面，Docker 更轻量，VM 占用的内存和磁盘更多。

在启动速度方面，Docker 启动更快，VM 需要启动完整操作系统，因此更慢。

在隔离性方面，VM 的隔离性通常更强，Docker 的隔离相对较弱。

在使用场景方面，Docker 更适合微服务和快速扩容，VM 更适合需要完整操作系统或强隔离的场景。



# talk about how to use docker in real project

1. In a real project, we usually start by writing a Dockerfile to define the application’s runtime environment, dependencies, files, and startup command.
2. For example, in a Java project, we can use a JDK image to build the application and a smaller JRE image to run the packaged JAR file.
3. We then use the `docker build` command to build a Docker image for the application.
4. After the image is built, we can use `docker run` to start a container locally and verify that the application works correctly.
5. Configuration such as server ports, database URLs, and Redis addresses is usually passed into the container through environment variables.
6. Sensitive information, such as passwords and API keys, should not be stored directly in the Dockerfile or image. It should be managed through secrets or external configuration.
7. For dependent services such as MySQL, Redis, Kafka, or RabbitMQ, we can use Docker Compose to manage them together in local development or testing environments.
8. Docker Compose uses a `docker-compose.yml` file to define and start multiple containers with one command.
9. After testing, we tag the image with a version number, such as `order-service:1.0.0`.
10. We then push the image to a container registry, such as Docker Hub, Harbor, Amazon ECR, or another cloud container registry.
11. In testing or production environments, the deployment system pulls the image from the registry and creates new containers based on that image. When the application is upgraded, we build and deploy a new image instead of manually changing the existing container.
12. In large-scale projects, Docker is usually used together with Kubernetes to support automated deployment, scaling, service recovery, load balancing, and rolling updates.



1. 在真实项目中，通常先为应用编写一个 Dockerfile，用来定义运行环境、依赖和启动命令。
2. 例如 Java 项目可以基于 JDK 或 JRE 镜像，把打好的 jar 包复制到镜像中。
3. 然后通过 `docker build` 命令构建项目对应的 Docker Image。
4. 构建完成后，可以先在本地使用 `docker run` 启动容器并验证应用是否正常。
5. 项目中的端口、数据库地址和 Redis 地址等配置，通常通过环境变量传入容器。
6. 对于 MySQL、Redis 和消息队列等依赖服务，可以使用 Docker Compose 统一管理。
   - Docker Compose 适合在一台机器上统一启动和管理多个容器，常用于本地开发、集成测试和小型项目部署。
   - 对于需要多节点部署、自动扩缩容、故障恢复、服务发现和滚动更新的大型生产环境，通常会使用 Kubernetes。
7. Docker Compose 可以通过一个 `docker-compose.yml` 文件同时启动多个容器。
8. 测试通过后，会给镜像打上版本标签，例如 `order-service:1.0.0`。
9. 然后把镜像推送到 Docker Hub、Harbor 或云厂商提供的镜像仓库。
10. 在测试或生产服务器上，从镜像仓库拉取镜像并启动对应容器。
11. 项目升级时，只需要构建新版本镜像并重新部署容器，不需要手动安装运行环境。
12. 在大型项目中，通常还会结合 Kubernetes，实现容器的自动部署、扩容、故障恢复和滚动更新。



# check the following AWS Services and how to use them in project

## EC2

Amazon EC2 provides scalable virtual servers in the AWS Cloud.

In a project, we can launch an EC2 instance and deploy a Java application, Docker container, or web server on it

EC2 is suitable when we need full control over the operating system, networking, storage, and application environment. 



EC2 可以理解为一台可远程管理的虚拟机。

项目中可以在 EC2 上安装 JDK、Docker、Nginx，并部署 Java 应用。

通常通过安全组开放 22、80、443 等必要端口，并使用 IAM Role 访问其他 AWS 服务。

它适合需要完全控制操作系统、网络和运行环境的项目。





## ECS

Amazon ECS is a fully managed container orchestration service for deploying and managing containerized applications.

In a project, we define ECS tasks and services to run Docker containers on EC2 instances or AWS Fargate.

ECS can automatically restart failed containers, scale the number of tasks, and integrate with load balancers.



ECS 是 AWS 提供的容器编排服务，用于部署和管理 Docker 容器。

项目中先定义 Task Definition，配置镜像、CPU、内存、端口和环境变量。

然后通过 ECS Service 运行多个容器实例，并结合负载均衡器对外提供服务。

ECS 可以运行在 EC2 或 Fargate 上，其中 Fargate 不需要自己管理服务器。



## ECR

Amazon ECR is a managed container registry used to store and manage Docker and OCI images.

In a project, the CI/CD pipeline builds an application image, tags it with a version, and pushes it to an ECR repository.

ECS, EKS, or EC2 can then pull the image from ECR when deploying the application.



ECR 是 AWS 提供的 Docker 镜像仓库，类似于 Docker Hub。

项目构建完成后，可以把 Java 应用打成 Docker Image 并推送到 ECR。

ECS、EKS 或 EC2 可以从 ECR 拉取镜像并启动容器。

实际项目中通常通过 CI/CD 自动完成镜像构建、打标签和推送。



## RDS

Amazon RDS is a managed relational database service that supports engines such as MySQL, PostgreSQL, Oracle, and SQL Server.

In a project, the backend application connects to an RDS instance to store structured data such as users, orders, and payments.

AWS manages tasks such as backups, software patching, failure detection, recovery, and optional high availability. 



RDS 是 AWS 托管的关系型数据库服务，支持 MySQL、PostgreSQL、MariaDB 等数据库。

项目中可以使用 RDS 存储用户、订单、支付等结构化业务数据。

Java 应用通过 JDBC 地址、用户名和密码连接 RDS，连接信息通常存放在 Secrets Manager 中。

RDS 可以提供自动备份、主从复制、故障切换和读写分离等能力。



## DocumentDB

Amazon DocumentDB is a fully managed document database service with MongoDB compatibility.

In a project, we can use MongoDB drivers to store flexible JSON-like documents such as product catalogs, user profiles, or content data.

It is useful when the data structure changes frequently and does not fit naturally into relational tables. 



DocumentDB 是 AWS 提供的文档型数据库，兼容部分 MongoDB API。

项目中适合存储 JSON 文档、商品属性、内容数据和结构变化频繁的数据。

Java 应用可以使用 MongoDB Driver 连接 DocumentDB，但需要确认功能兼容性。

DocumentDB 由 AWS 管理集群、备份和高可用，但通常需要部署在 VPC 内访问。



## DynamoDB

Amazon DynamoDB is a serverless and fully managed NoSQL database that supports key-value and document data models.

In a project, we can use it to store shopping carts, sessions, game states, order statuses, or other high-traffic data.

It is suitable for applications that require automatic scaling and consistently low-latency access at a large scale. 



DynamoDB 是 AWS 提供的无服务器键值和文档数据库。

项目中适合存储访问量大、延迟要求低的数据，例如会话、购物车和设备状态。

使用时需要合理设计 Partition Key 和 Sort Key，避免出现热点分区。

Java 项目可以通过 AWS SDK 读写 DynamoDB，并按请求量或容量模式计费



## Lambda Function

AWS Lambda allows us to run code without provisioning or managing servers.

In a project, a Lambda function can be triggered by API Gateway, SQS, SNS, DynamoDB Streams, or other AWS events.

It is useful for short, event-driven tasks such as processing files, sending notifications, or handling API requests.



Lambda 是 AWS 提供的无服务器函数计算服务，可以在事件触发时执行代码。

项目中可以用它处理文件上传、定时任务、消息消费和简单 API 请求。

例如用户上传文件到 S3 后，可以自动触发 Lambda 生成缩略图。

Lambda 不需要管理服务器，但需要注意执行时间、内存、冷启动和并发限制。



## API gateway

Amazon API Gateway is used to create, publish, secure, monitor, and manage REST, HTTP, and WebSocket APIs.

In a project, it acts as the entry point that receives client requests and forwards them to Lambda, EC2, ECS, or other backend services.

It can also handle authentication, authorization, traffic control, monitoring, and API version management.



API Gateway 是 AWS 提供的 API 入口和管理服务。

项目中可以通过它对外暴露 REST API、HTTP API 或 WebSocket API。

API Gateway 可以把请求转发到 Lambda、EC2、ECS 或其他后端服务。

它还可以处理认证、限流、日志、跨域和请求参数转换。



## AWS Kinesis

Amazon Kinesis is used to collect, process, and analyze streaming data in real time.

In a project, Kinesis Data Streams can receive continuous events such as application logs, user clicks, transactions, or IoT data.

Consumer applications can process these events for real-time monitoring, analytics, or event-driven workflows. 



Kinesis 是 AWS 提供的实时数据流处理服务。

项目中可以用它收集用户行为日志、交易事件、设备数据和应用日志。

生产者持续向 Kinesis Stream 写入数据，消费者可以使用 Lambda 或应用程序实时处理。

它适合高吞吐、持续产生并且需要实时分析的数据场景。



## IAM

AWS IAM controls who can access AWS resources and what actions they are allowed to perform.

In a project, we create IAM roles and policies to give applications, developers, and AWS services only the permissions they need.

For example, an ECS task can use an IAM role to access an S3 bucket without storing permanent AWS credentials in the application.



IAM 是 AWS 的身份认证和权限管理服务。

项目中可以创建 User、Group、Role 和 Policy，控制谁可以访问哪些 AWS 资源。

EC2、ECS 和 Lambda 通常通过 IAM Role 访问 S3、SQS、DynamoDB 等服务。

权限配置应遵循最小权限原则，避免直接使用 Root 用户或长期 Access Key。





## SNS, SQS

Amazon SNS is a fully managed publish-subscribe messaging service.

In a project, a service publishes an event to an SNS topic, and SNS delivers copies of the event to multiple subscribers such as SQS queues, Lambda functions, email addresses, or HTTP endpoints.

It is commonly used for notifications and fan-out communication between multiple services. 



Amazon SQS is a fully managed message queue used to decouple application components.

In a project, one service sends messages to a queue, while another service processes those messages asynchronously.

It helps handle traffic spikes, retry failed operations, and prevent one service’s temporary failure from directly affecting another service.



SNS 是消息发布订阅服务，SQS 是消息队列服务。

SNS 适合把一条消息同时推送给多个订阅者，而 SQS 适合异步排队和削峰。

项目中可以让订单服务向 SNS 发布订单事件，再由多个 SQS 队列分别接收消息。

消费者从 SQS 拉取消息处理，并结合重试机制和死信队列处理失败消息。





# design online shopping system, only use aws services and draw the architecture

用户通过 Route 53 解析域名，并由 CloudFront 统一接收网站访问请求。

React、HTML、CSS、JavaScript 和商品图片等静态资源存储在 Amazon S3 中，并通过 CloudFront 加速分发。

用户发送的 API 请求由 CloudFront 转发到 API Gateway，API Gateway 负责统一入口、限流和请求验证。

用户注册、登录和身份认证使用 Amazon Cognito，API Gateway 负责验证 Cognito 生成的 Token。

API Gateway 通过 VPC Link 访问私有的 Application Load Balancer，再由 ALB 将请求分发到不同的 ECS Fargate 微服务。

用户、商品、购物车、订单、库存和支付等服务以 Docker 容器运行，容器镜像存储在 Amazon ECR 中。

用户、订单、支付和库存等需要事务保证的结构化数据可以存储在 Amazon Aurora 或 RDS 中。

商品目录和动态商品属性可以存储在 DocumentDB，购物车、会话和幂等记录可以存储在 DynamoDB。

ElastiCache for Redis 可以用于缓存热门商品、限流和临时库存预留，从而降低数据库压力。

创建订单后，订单服务将事件发布到 SNS，再由库存、支付和通知服务对应的 SQS 队列异步处理。

Kinesis 用于收集用户点击、搜索和购买行为，Lambda 和 EventBridge 可以处理图片压缩、定时任务和订单超时取消。

CloudWatch、X-Ray、IAM、Secrets Manager、KMS 和 CloudTrail 分别负责监控、链路追踪、权限控制、密钥管理、数据加密和操作审计。
